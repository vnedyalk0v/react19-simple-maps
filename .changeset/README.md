# Changesets

This project uses Changesets for package versioning, release pull requests, and GitHub release text.

When a task requires a new `.changeset/*.md` file, follow the shared release-writing rules in
`RELEASE_NOTES_GUIDELINES.md`.

Key rules:

- Write in plain English.
- Do not use emojis, commit hashes, or internal tool names.
- Describe the user-visible package impact.
- Keep the first sentence strong enough to stand on its own in release notes.
